# 案件事实-关系图谱 典型样例与验收用例

## 1. 文档目标
本文档用于为产品评审、算法验证、测试验收和试点上线提供统一样例。核心目的是明确：

- 哪些案件场景应优先覆盖
- 系统应抽出哪些实体、关系、属性和来源
- 图谱页面应如何展示
- 什么结果可判定为“首版可用”

## 2. 验收原则
- 抽取结果不要求一次性完全正确，但必须可回溯、可校核
- 系统不得把“当事人主张”直接当作“法院认定”
- 复杂关系场景应优先保证关键关系不漏抽
- 对冲突关系必须显式标记，不能静默覆盖
- 页面展示应服务法官理解案件，而不是只展示算法结果

## 3. 典型案件样例

### 3.1 样例一：民间借贷 + 保证担保

#### 输入背景
- 原告张三起诉称，2024 年 3 月 1 日向被告李四出借 50 万元
- 被告王五在借款合同上签字，承诺承担连带责任保证
- 原告提交借款合同、转账记录、保证书
- 被告李四答辩称，该 50 万元并非借款，而是合作投资款

#### 期望抽取实体
- 张三：自然人，原告，出借人
- 李四：自然人，被告，借款相对方
- 王五：自然人，被告或第三人，保证人
- 借款合同：合同类虚拟实体
- 转账记录：证据材料
- 保证书：证据材料

#### 期望抽取关系
- 张三 --出借--> 李四
- 王五 --保证担保--> 李四
- 张三 --提交证据--> 借款合同
- 张三 --提交证据--> 转账记录
- 李四 --否认关系--> 借贷关系
- 李四 --主张关系--> 投资关系

#### 期望抽取属性
- 借款金额：500000 元
- 借款时间：2024-03-01
- 担保方式：连带责任保证
- 来源文书：起诉状、答辩状、借款合同、保证书

#### 页面展示要求
- 在资金关系视图中应突出张三、李四、王五三方关系
- “借贷关系”和“投资关系”应并列标记为争议
- 点击保证关系可回看保证书内容

### 3.2 样例二：婚姻家事 + 夫妻共同财产

#### 输入背景
- 原告赵某起诉离婚，请求分割婚后共同财产
- 双方争议涉案房产是否属于夫妻共同财产
- 被告称房产由其父母出资购买，登记在其名下

#### 期望抽取实体
- 原告赵某
- 被告钱某
- 钱某父亲
- 钱某母亲
- 涉案房产
- 房产证或购房合同

#### 期望抽取关系
- 赵某 --配偶--> 钱某
- 钱某 --所有/登记在名下--> 涉案房产
- 钱某父母 --出资--> 涉案房产或购房事件
- 赵某 --主张关系--> 夫妻共同财产
- 钱某 --否认关系--> 夫妻共同财产

#### 页面展示要求
- 人物关系视图中可直观看到夫妻与父母关系
- 涉案财产视图中可展示房产归属、登记、出资三类关系
- 对“是否属于共同财产”应标记为争议关系

### 3.3 样例三：公司控制 + 股权代持

#### 输入背景
- 原告甲主张其系某公司实际出资人，被告乙仅为名义股东
- 双方存在代持协议
- 公司工商登记显示乙持股 60%

#### 期望抽取实体
- 原告甲
- 被告乙
- 某公司
- 股权份额 60%
- 代持协议
- 工商登记材料

#### 期望抽取关系
- 乙 --股东--> 某公司
- 甲 --代持--> 股权份额
- 股权份额 --对应公司--> 某公司
- 甲 --主张关系--> 实际出资人
- 工商登记材料 --支撑--> 股东关系

#### 页面展示要求
- 公司控制视图中同时显示名义股东和实际控制关系
- 代持关系应可查看协议来源
- 工商登记关系与代持主张不可混淆

### 3.4 样例四：执行案件 + 财产代持

#### 输入背景
- 申请执行人主张被执行人将房产、车辆登记在亲属名下，规避执行
- 案外人提出执行异议，称财产归其所有

#### 期望抽取实体
- 申请执行人
- 被执行人
- 案外人
- 房产
- 车辆

#### 期望抽取关系
- 被执行人 --控制财产--> 房产
- 被执行人 --控制财产--> 车辆
- 案外人 --所有--> 房产/车辆
- 案外人 --提出异议--> 执行程序

#### 页面展示要求
- 涉案财产视图中支持查看控制链和名义归属
- 争议关系需要明显标记

### 3.5 样例五：刑事案件 + 共同犯罪

#### 输入背景
- 起诉书指控甲、乙、丙共同实施诈骗
- 甲负责联系被害人，乙负责收款，丙负责转移资金

#### 期望抽取实体
- 被告人甲、乙、丙
- 被害人
- 涉案账户
- 诈骗所得

#### 期望抽取关系
- 甲 --共犯--> 乙
- 乙 --共犯--> 丙
- 乙 --收款--> 涉案账户
- 丙 --转移资金--> 涉案账户或赃款
- 甲 --实施行为--> 联系被害人

#### 页面展示要求
- 刑事扩展场景中应支持行为链与资金流向联动

### 3.6 样例六：行政案件 + 行政行为链

#### 输入背景
- 某行政机关对原告作出行政处罚决定
- 原告不服，提起行政诉讼

#### 期望抽取实体
- 原告
- 行政机关
- 行政处罚决定书
- 复议决定书

#### 期望抽取关系
- 行政机关 --作出行政行为--> 行政处罚决定书
- 行政处罚决定书 --指向--> 原告
- 原告 --提起诉讼--> 案件

#### 页面展示要求
- 行政扩展场景中应支持行为链条和送达链条展示

## 4. 三元组验收样例

### 4.1 基础三元组示例
```json
{
  "head": "张三",
  "relation": "出借",
  "tail": "李四",
  "attributes": {
    "amount": 500000,
    "time": "2024-03-01"
  },
  "source": {
    "documentType": "起诉状",
    "sourceParty": "原告"
  }
}
```

### 4.2 冲突三元组示例
```json
[
  {
    "head": "张三",

    "tail": "李四",
    "status": "有争议",
    "sourceParty": "原告"
  },
  {
    "head": "张三",
    "relation": "投资",
    "tail": "李四",
    "status": "有争议",
    "sourceParty": "被告"
  }
]
```

## 5. 功能验收用例

### 5.1 文书解析验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| F01 | 可识别文书类型 | 起诉状、答辩状、上诉状识别正确 |
| F02 | 可切分主要结构区块 | 当事人信息区、事实理由区可拆出 |
| F03 | 可保留段落定位 | 页面可回看原文段落 |

### 5.2 实体抽取验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| E01 | 主体识别 | 原告、被告、第三人、代理人可识别 |
| E02 | 实体类型细分 | 自然人、组织、账户、财产可区分 |
| E03 | 实体归一化 | 同一主体多写法可合并或提示合并 |

### 5.3 关系抽取验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| R01 | 基础法律关系抽取 | 借贷、担保、配偶、法定代表人等可抽出 |
| R02 | 关系属性抽取 | 金额、时间、合同编号可挂到关系上 |
| R03 | 冲突关系保留 | 原告与被告不同主张可并列展示 |
| R04 | 关系可回溯 | 点击关系可回到文书来源 |

### 5.4 页面交互验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| U01 | 图谱可加载 | 可正常展示节点和边 |
| U02 | 可切换视图 | 人物、资金、控制、财产视图可切换 |
| U03 | 可筛选争议关系 | 可单独查看争议关系 |
| U04 | 可查看详情 | 节点和边点击后右侧详情正确展示 |

### 5.5 人工校核验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| H01 | 实体可确认 | 可将系统结果标记为已确认 |
| H02 | 实体可合并/拆分 | 操作后图谱正确更新 |
| H03 | 关系可修改 | 关系类型和属性可编辑 |
| H04 | 校核有日志 | 操作人、时间、前后变化可查看 |

### 5.6 版本管理验收
| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| V01 | 重跑生成新版本 | 不覆盖旧版本 |
| V02 | 可查看默认版本 | 默认办案视图展示当前发布版本 |
| V03 | 可追踪人工校核版本 | 校核结果形成独立版本 |

## 6. 试点验收建议

### 6.1 案件样本建议
首批建议每类场景选择 10-20 件典型案件：
- 民间借贷
- 婚姻家事
- 继承
- 公司控制/股权代持
- 担保追偿

### 6.2 验收方式建议
- 产品演示验收
- 结构化抽取质量抽检
- 法官试用反馈
- 人工校核工作量统计

### 6.3 首版可用判定标准
满足以下条件即可判定首版达到可试点水平：
- 关键主体和关键关系可自动生成
- 主要关系可回溯到来源
- 争议关系可并列展示
- 人工校核可以闭环修正结果
- 法官能够通过图谱比传统阅卷更快建立案件理解

## 7. 结论
案件事实-关系图谱的验收，不应只看“抽了多少三元组”，而应看是否真正帮助法官理解案件。首版最应关注的不是全覆盖，而是：

- 关键案件场景是否打透
- 关键关系是否清楚
- 来源是否可信
- 校核是否可闭环

只要这四点成立，产品就具备进入法院试点的基础。
